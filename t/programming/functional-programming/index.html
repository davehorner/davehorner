    Functional programming

    Lazy functional programming languages and the programmer's who think in terms of functions, compositions, and transformations.
    with claims like formal provability, modularity, composability, and an apparent fallout of easier debugging and testing from compact side effect free code.

    There are ways of doing functional programming using common programming languages that you might not originally consider; I've mentioned libraries for functional programming languages in Javascript/Perl/Python/etc pages on my site. "functions as first-class objects” and “eliminating side effects” don't happen automatically, some languages make it easier to implement than others. immutable and stateless programs require a different train of thought when attacking the problem.
